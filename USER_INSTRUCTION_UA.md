# ІНСТРУКЦІЯ КОРИСТУВАЧА — LinkFlow (актуально)

## Встановлення

### З Google Play Store
- Відкрийте Google Play Store
- Знайдіть "LinkFlow"
- Натисніть "Встановити"

### З APK файлу
- Завантажте файл `app-release.apk`
- Дозвольте встановлення з невідомих джерел у налаштуваннях
- Відкрийте файл і встановіть

---

## Перший запуск
1. Відкрийте застосунок
2. На головному екрані натисніть "Налаштування"
3. Пройдіть біометричну автентифікацію (Face ID / Touch ID / пароль пристрою)
4. Потрапите на екран налаштувань/проєктів

---

## Налаштування проєктів

### Створення
1. На екрані "Проєкти" натисніть "+"
2. Заповніть поля:
   - **Назва** — довільна назва, наприклад "Проєкт 1"
   - **Regex** — регулярний вираз лише для змінної частини (без `reich://`)
     - Приклад (рекомендовано): `^(\d+)$`
   - **URL шаблон** — адреса для відкриття, де `{key}` буде підставлено зі співпадіння
     - Приклад: `https://example.com/item/{key}`
3. Натисніть "Зберегти"

### Тестування
1. На екрані редагування проєкту введіть тестовий рядок QR (можна з або без `reich://`):
   - Наприклад: `reich://123` або просто `123`
2. Натисніть "Тестувати"
3. Якщо regex валідний і є рівно одне співпадіння з групою — побачите сформований URL

### Редагування / Видалення
- Редагування: натисніть на проєкт у списку → внесіть зміни → "Зберегти"
- Видалення: меню (⋮) праворуч → "Видалити" → підтвердіть

---

## Робота з QR-кодами
- Додаток обробляє посилання зі схемою `reich://` (реєстрація у системі Android).
- При скануванні валідного QR-коду відкривається відповідний URL у браузері.
- При невалідному QR-коді з’являється алерт з поясненням та кнопкою повернення на головний екран.

---

## Приклади налаштувань (оновлено)

### Приклад 1: Простий числовий ключ
- Regex (без схеми): `^(\d+)$`
- URL: `https://example.com/item/{key}`
- QR: `reich://123` або `123` → відкриється `https://example.com/item/123`

### Приклад 2: Ключ з префіксом "1"
- Regex: `^1(\d+)$`
- URL: `https://special.example.com/item/{key}`
- QR: `reich://1234` → захопить `234` → `https://special.example.com/item/234`

### Приклад 3: Шлях з сегментом
- Regex: `^project/(\d+)$`
- URL: `https://example.com/project/{key}`
- QR: `reich://project/456` → `https://example.com/project/456`

---

## Важливі моменти
- Regex має давати рівно одне співпадіння.
- Повинна бути принаймні одна група захоплення `( ... )` — остання група підставляється як `{key}`.
- Для точного співпадіння додавайте `^` (початок) та `$` (кінець).
- Схему `reich://` у regex НЕ додавайте — додаток автоматично пробує матчити як повний лінк, так і «хвіст» без схеми.
